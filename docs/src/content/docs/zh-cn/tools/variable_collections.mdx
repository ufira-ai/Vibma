管理变量集合和模式。

<details>
<summary>Type reference</summary>

```ts
// Variable collections group related design tokens (variables) and define their modes.
// Modes represent different contexts (e.g. Light/Dark theme, Desktop/Mobile breakpoint). Each collection has 1+ modes.
// Workflow: create collection → add_mode for extra modes → create variables in the collection.
// Deleting a collection cascades — all variables inside it are also deleted.
// The default mode always exists and cannot be removed. Use add_mode/remove_mode for additional modes.
// Collection IDs look like "VariableCollectionId:0:1". Mode IDs look like "0:0" or "1:0".
interface Collection { id: string; name: string; modes: {modeId: string, name: string}[]; defaultModeId: string; variableNames: string[] }
```

</details>
