管理设计变量（颜色、数值、字符串、布尔）。

<details>
<summary>Type reference</summary>

```ts
// Variables are design tokens — reusable values (colors, numbers, strings, booleans) that can be bound to node properties.
// Variables are identified by name (unique within a collection). No IDs needed — use name to get/update/delete.
// Workflow: create a variable_collection first → then create variables inside it → bind to nodes via frames/text update PatchItem.bindings.
// ---
// Each variable has values per mode (e.g. Light/Dark). Use modeId when setting values for a specific mode.
// valuesByMode keys are mode IDs (from the collection), values match the variable's resolvedType.
// Aliases: a variable can reference another variable by name via {type: "VARIABLE_ALIAS", name: "other/variable"}.
// scopes: restrict where variables appear in the UI. Values: "ALL_SCOPES", "TEXT_CONTENT", "WIDTH_HEIGHT", "GAP",
//   "CORNER_RADIUS", "ALL_FILLS", "FRAME_FILL", "SHAPE_FILL", "TEXT_FILL", "STROKE_COLOR", "STROKE_FLOAT",
//   "EFFECT_FLOAT", "EFFECT_COLOR", "OPACITY", "FONT_FAMILY", "FONT_STYLE", "FONT_WEIGHT", "FONT_SIZE", "LINE_HEIGHT", "LETTER_SPACING", "PARAGRAPH_SPACING", "PARAGRAPH_INDENT"
interface Variable { name: string; resolvedType: "COLOR"|"FLOAT"|"STRING"|"BOOLEAN"; collectionName: string; valuesByMode: Record<string, number | boolean | string | Color | {type: "VARIABLE_ALIAS", name: string}> }
```
